# Тестовая схема yogood_tst

## Структура репозитория

1. `ddl.sql` – создание схемы `yogood_tst` и всех таблиц:
   - units, categories, products, product_types, fillers, volumes
   - order_statuses, deliveries, orders, order_items
   - cancellation_reasons, canceled_orders, product_prices

2. `insert.sql` – загрузка справочников и тестовых данных:
   - единицы измерения, категории, продукты, типы, наполнители, объемы
   - статусы заказов и причины отмены
   - тестовые заказы клиента "Муся Кукуся" с доставкой и самовывозом

3. `select_order.sql` – пример запроса для получения полной информации по конкретному заказу:
   - данные клиента, статус, доставка
   - продукты с типом, наполнителем, объемом или весом, количеством

## Особенности реализации

- Заказ (`orders`) связан с несколькими продуктами через таблицу `order_items` (связь 1:N)
- Используются справочники для типизации и стандартизации:
  - тип продукта (`product_types`), наполнитель (`fillers`), единицы измерения (`units`), объем (`volumes`)
  - статусы заказов (`order_statuses`), причины отмены (`cancellation_reasons`)
- Возможны разные сценарии:
  - доставка (`is_delivery = TRUE`) с таблицей `deliveries`
  - самовывоз (`is_delivery = FALSE`)
- Возможность отмены заказа с указанием причины через таблицу `canceled_orders`
